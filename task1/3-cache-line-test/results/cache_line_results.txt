============================================
  CPU Cache Line 大小验证测试
  数组大小: 32 MB
============================================

|  Stride(B) | 总时间(ns) | 相对速度(越大越快) |
|------------|--------------|----------------------|
|          1 |     10162964 |                 1.00 |
|          2 |      5612435 |                 1.81 |
|          4 |      4143695 |                 2.45 |
|          8 |      3305190 |                 3.07 |
|         16 |      2994348 |                 3.39 |
|         32 |      2890062 |                 3.52 |
|         64 |      2496608 |                 4.07 |
|        128 |      1694411 |                 6.00 |
|        256 |       808659 |                12.57 |
|        512 |       516116 |                19.69 |

结果解读：
  - strides=1 最快（同一 Cache Line 内连续访问）
  - stride=64 附近出现拐点（Cache Line = 64 字节）
  - stride>64 后速度趋于稳定（每次都跨越 Cache Line 边界）

建议配合 perf 使用：
  sudo perf stat -e L1-dcache-load-misses,LLC-load-misses ./cache_line_test --stride 1
  sudo perf stat -e L1-dcache-load-misses,LLC-load-misses ./cache_line_test --stride 64
